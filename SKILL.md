# 半秒第一反应改写

这是一个中文单文件 skill。它只做一件事：把用户的某个自动反应，改写成一句能反复安装的动作脚本。

方法主要来自《The Half Second》第七章。ARISE 是主干；其他章节只用来解释为什么这样写、怎么重复、哪里不能用。

用户一说“我总是一刺激就……”“我明知道但做不到”“我想改掉这个反应”，先别讲理论。先把那半秒抓出来。

---

## 工作习惯

先问动作，不问人格。用户说“我太没自制力”，你要追问：“那一瞬间你做了什么？”

先写句子，不写讲义。除非用户追问原理，否则解释到够用就停。

脚本要短。短到能在走路、刷牙、等电梯时顺口念出来。越像海报，越危险；越像用户自己平时会说的话，越好。

如果脚本让用户觉得有点傻，可以继续。如果它让用户更羞耻、更假、更想辩解，重写。

安全边界放在前面。创伤、化学依赖、精神危机、虐待关系、现实威胁，不要用脚本硬扛。

---

## 快速流程

把问题抓成这四项：

- 触发线索：什么时候旧反应启动？
- 旧动作：半秒里已经做了什么？
- 新动作：希望半秒先做什么？
- 范围：这句是普遍式，还是只在某个线索下成立？

例子：

- 触发线索：账单来了。
- 旧动作：关掉，不看。
- 新动作：打开账单。
- 范围：情境式。

脚本可以是：

`账单一到，我 60 秒内打开。`

如果用户会强烈抵抗，可以加一句理由：

`账单一到，我 60 秒内打开，因为看见才有选择。`

---

## ARISE 怎么用

ARISE 是五个设计维度，不是五个填空题。

| 维度 | 用法 |
|---|---|
| Action 动作 | 必填。半秒里要执行的行为。小、具体、能开始。 |
| Reason 理由 | 可选。意识会抬杠时，加一句短、真、具体的“因为”。 |
| Identity 身份 | 通常藏在动作里。能不写“我是……”就不写。 |
| Situation 情境 | 可选。只有情境式脚本才写“当 X 时”。 |
| Emotion 情绪 | 可选。真实情绪可以放进词、理由、记忆或语气里。别演。 |

每句脚本还要过三关：

- 现在时，不写“我将会”。
- 一口气说完，最好 3 到 5 秒。
- 像用户自己的话，不像名言、口号或咨询手册。

---

## 写脚本

先定 Action。半秒能做动作，不能做心情。

可用：

- `我吸三口气，问一个问题。`
- `数字跳动时，我等一天再决定。`
- `她说话时，我先看她眼睛再回答。`
- `错误出现时，我只找下一个可改动作。`

不可用：

- `我保持冷静。`
- `我不焦虑。`
- `我变得自律。`
- `我以后一定不拖延。`

测试很简单：外人能不能看出它发生了？看不出，就继续改。

然后定范围。

普遍式适合整类场景都成立的句子：

- `我不抽烟。`
- `我回应，不反应。`
- `我按周判断，不按分钟判断。`

情境式适合具体线索：

- `账单一到，我 60 秒内打开。`
- `咖啡放下后，我打开文档写第一句。`
- `数字跌过阈值时，我等一天再决定。`

能诚实普遍化才普遍化。普遍式重复机会多，装得快，但别拿它压掉正当愤怒、安全警报或真实偏好。

Identity 通常不用明写。`我不抽烟` 已经带着“非吸烟者”；`我回应，不反应` 已经带着“回应者”。如果动作太泛，先打磨动作，最后才考虑加一句 `我是...`。不要写 `因为我是一个非吸烟者，所以我不抽烟`，这会把身份变成辩题。

Reason 只在需要时加。好理由经得起第 1000 遍：

- `因为看见才有选择。`
- `因为同一个数字明天看会不一样。`
- `因为羞辱不能修正问题。`

假理由宁可删掉。`因为我要成为最好的自己` 太大，`因为宇宙会奖励我` 除非用户真的信，否则会坏掉。

Emotion 可以加速，但只能用真的。可以在词里，在理由里，在一个具体记忆里，也可以在说话的语气里。安静地确定，比假装热血好。

---

## 路由检查

让用户念一遍草稿，观察身体反应。

| 反应 | 处理 |
|---|---|
| 有点傻、陌生、好笑 | 可以继续。 |
| 更羞耻、更假、像撒谎 | 重写。 |
| 马上开始辩解 | 缩短，换动作，删掉或改掉理由。 |
| 像别人写的口号 | 改成用户自己的话。 |
| 念不完、拗口 | 压短。 |
| 念完更稳一点 | 可以进入安装。 |

常见替换：

| 原句 | 改成 |
|---|---|
| 我很平静 | 我吸三口气，问一个问题 |
| 我很富足 | 账单一到，我 60 秒内打开 |
| 我不再焦虑 | 数字跳动时，我等一天 |
| 我会更自律 | 咖啡放下后，我打开文档写第一句 |
| 我不发火 | 她说话时，我先看她眼睛再回答 |

## 安装

脚本写好后，要给用户一个现实的安装计划。

随处重复。走路、洗澡、开车、睡前、等水烧开，都可以。触发场景只是提醒，不是唯一训练场。只等到触发时才念，通常太晚。

前几十遍到上百遍，觉得傻很正常。后来变成“怎么又是这句”的烦，反而可能是进展。脚本开始自己冒出来时，说明它已经在抢位置了。

一次只跑一两句。按 3 到 12 个月看，不用两周就下结论。稳定之后，低频维护就够了。旧反应偶尔回来，不等于失败，只是提醒你补几次。

同时改环境。烟别放抽屉里，手机别放枕边，交易软件别放第一屏，账单别藏到看不见的地方。环境一直在写，脚本不是唯一的笔。

---

## 调参

脚本太假，通常是目标太大、身份太早、理由不真，或者话不像用户自己说的。降到一个能做的小动作，删掉多余身份，换成更朴素的话。

脚本太弱，可能是动作抽象、线索不清、缺少真实理由。把动作写得更具体；必要时改成情境式。

高压下想不起来，就压短。也可以改人称：

- `我先吸气，再回答。`
- `你先吸气，再回答。`
- `小林，先吸气，再问一句。`

如果安装后更不像自己，停。脚本可能盖住了真实偏好或边界。重新问：我真的愿意成为会做这个动作的人吗？

---

## 调音杠杆

脚本基本对，但就是装不进去时，先别急着换目标。调这三件事。

人称。默认用第一人称：`我先吸气，再回答。` 高压下第一人称有时会被旧反应劫持，可以换第二人称：`你先吸气，再回答。` 如果还不够，用自己的名字制造一点距离：`小林，先吸气，再问一句。`

语言。旧声音如果是用母语装进去的，比如童年的内在批评，试试第二语言。不是为了显得高级，而是换一条路，避开旧声音的主场。

声音。一个人时，出声念通常更有分量；不方便就默念。别演讲，别表演情绪。像对自己说一句真话就够了。

## 场景

拖延：目标不是“我自律”，而是启动。

- `咖啡放下后，我打开文档写第一句。`
- `坐到桌前，我先做两分钟。`

内在批评：目标不是“我足够好”，而是把攻击换成修正。

- `错误出现时，我只找下一个可改动作。`
- `旧错误冒出来时，我回到今天这一件事。`

这里要说清楚一件事：目标是重新排序，不是让旧声音闭嘴。旧声音几个月后可能还会冒出来；成功的样子通常是新句子先到，旧声音变成背景音。用户如果期待它彻底消失，下一次旧声音出现时很容易以为自己失败，然后放弃。

还要把标准和批评者分开。有用的标准是具体的，指向一个改法，比如“这一段还要再过一遍”。批评者是全局的、羞辱性的，比如“你什么都做不成”。只对批评者写脚本；标准留着。

关系冲突：目标不是“不生气”，而是让第一动作变小。

- `她说话时，我先看她眼睛再回答。`
- `声音升高时，我先吸气，问一句具体问题。`

只处理不成比例的反应。别把真实冒犯、危险信号或该设边界的愤怒脚本掉。如果关系里有控制或虐待，一句“更冷静”的脚本可能会训练用户更顺地吸纳伤害。那时目标是安全、求助、离开或设边界，不是把反应磨得更温顺。

金钱：目标不是“我很富足”，而是看见数字后做一个动作。

- `账单一到，我 60 秒内打开。`
- `数字跌过阈值时，我等一天再决定。`
- `我按周判断，不按分钟判断。`

轻中度习惯：可以用身份型普遍式，但先确认没有医学风险。

- `我不抽烟。`
- `饭后，我把杯子洗掉。`

学习后不行动：把理解变成一个当日动作。

- `读完一章，我写一句要安装的话。`
- `学到方法后，我今天只做第一个动作。`

## 为什么这样做

用户追问原理时，简短解释：

很多反应在深思熟虑之前已经启动。意志力不是没用，只是来得晚。半秒系统更像熟悉度计数器：它先问“这句我听过多少次”，不先审论文。

广告、家庭语言、平台、旧经历都在写第一反应。ARISE 只是让用户自己也开始写。动作句比心情句好，因为动作能执行；心情只能等它发生。理解也不等于安装。会解释，不代表半秒会做。

---

## 失败排查

脚本没用时，按这个顺序查：

1. 是不是在写感受，而不是动作？
2. 动作是不是太大？
3. 普遍式是不是不诚实？
4. 身份是不是太早？
5. 理由是不是假的？
6. 句子是不是不像用户本人？
7. 重复时间是不是太短？
8. 环境是不是还在高频写回旧反应？
9. 目标是不是越界了？

---

## 不能硬用的地方

这些情况不要只靠脚本：

- 严重或活动期创伤、复杂 PTSD、解离。
- 酒精、阿片类、苯二氮䓬类等化学依赖。
- 活动期躁狂、精神病性症状、重度抑郁、自伤风险。
- 虐待、控制、威胁、现实安全风险。
- 乐器、竞技动作等复杂运动技能。
- 脚本让用户更不像自己，或压掉真实偏好。

遇到这些，先处理安全、医疗、治疗和支持系统。ARISE 可以辅助，不能顶替。

## 输出模板

默认这样答：

```markdown
我先把它抓成半秒格式：

- 触发线索：
- 旧动作：
- 新动作：
- 范围：

脚本候选：
1. ...
2. ...

我会先用第 ... 句。原因是...

路由检查：
念起来如果只是有点傻，可以继续；如果更难受、像撒谎，我们改掉 ...

安装计划：
接下来 ... 周/月，只跑这一句。任何想起来的时候都重复，不只等触发时。前期觉得傻正常；变成“又是这句”的烦，可能是进展。
```

用户只要一句，就短一点：

```markdown
先用这句：

「...」

它的动作是 ...，范围是 ...。先别加太多东西，短一点更容易装进去。念起来只是有点傻就继续；如果更难受，我们再改。
```

最后记住：好脚本通常很朴素，甚至有点不好意思。它不负责让用户当场感觉伟大。它只负责在旧反应到来之前，递给半秒一个新动作。
